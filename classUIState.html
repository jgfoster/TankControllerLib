◆ sleepHandlers

SleepHandler UIState::sleepHandlers
staticprivate

SleepHandler

When running on the real Arduino, the delay() function will actually pause. When running on the mock Arduino, the delay() function increments the clock and then returns immediately. While this generally the right thing to do, this presents a couple issues. First, if we want to look at the state during the delay, this is difficult since the state could change after the delay (e.g., show a message on the LCD for one second, then clear it). Second, if we are using the mock Arduino library to emulate the application in a GUI, we want to have a real delay. To solve this, we will allow code (such as tests and the GUI) to register to be notified of delays. Then we can take whatever action is appropriate.

typedef void (*SleepHandler)(int ms);
